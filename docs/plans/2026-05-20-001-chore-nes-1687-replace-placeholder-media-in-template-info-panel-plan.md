# chore: NES-1687 replace placeholder media in template info panel

## Summary

Swap the seven media assets used by the `TemplateInfoPanel` side panel (added in NES-1538) for the final GIFs and PNGs now that all source files are available. Files are dropped into the existing `apps/journeys-admin/public/assets/template-info/` folder under the exact filenames the components and specs already reference, so no component code changes are required if the filenames are preserved.

## Problem Frame

NES-1538 shipped the Template Info side panel with a mix of real screenshots and placeholder PNG-bytes-as-GIF stubs (generated by `apps/journeys-admin/scripts/generate-template-info-placeholders.mjs`). The four `.png` slots currently appear to hold real content (518KB–1.5MB), while the three `.gif` slots are still ~4KB placeholders. With every real asset now available, the panel needs its final media in place; everything else about the panel — layout, dimensions, specs, copy — is already correct.

## Requirements

- R1. Every slot referenced by `TemplateInfoPanel` resolves to the final, intentional asset (no placeholder bytes).
- R2. Existing component specs continue to pass without modification (they assert exact `src` paths).
- R3. The placeholder generator script is no longer needed once real assets are in place — either remove it or document why it stays.

## Scope Boundaries

- Not changing any `TemplateInfoPanel` component code, copy, layout, or dimensions.
- Not introducing a new asset pipeline, CDN swap, or `next/image` migration.
- Not touching other help/info surfaces (e.g., `HelpScoutBeacon`, `TemplateSettingsDialog`).

### Deferred to Follow-Up Work

- Optimizing PNG file sizes (`template-types-regular.png` is currently 1.5MB) — deferred unless the new assets ship at similar sizes; flag in PR if size is a concern.

## Key Technical Decisions

- **Preserve existing filenames** rather than renaming to whatever the source files are called. Renaming would force parallel edits in 4 components, 4 specs, and the storybook — none of which add value. The mapping step (user-supplied filename → slot filename) happens during upload.
- **Delete the placeholder generator script** (`generate-template-info-placeholders.mjs`) once final assets are in. Its only purpose was to produce the stubs being replaced; leaving it adds clutter and confusion. (See U2.)
